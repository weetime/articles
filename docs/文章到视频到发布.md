# 文章 → 视频 → 发布:完整手册

把一篇技术报告做成动画解说视频,并发到各平台。两个 skill 配合:
- **report-to-video** —— 文章 → 视频(代码驱动动画 + 配音 + BGM)
- **publish-video** —— 视频 → YouTube / X(凭证集中,跨目录复用)

```
文章 article.md
   │  report-to-video(Remotion 动画 + edge-tts 配音 + 合成 BGM)
   ▼
video-portrait.mp4 (竖屏 9:16,默认产物)   [+ video-landscape.mp4 横屏,按需]
   │  publish-video
   ├─► YouTube  ── 横屏正片(API 上传)
   ├─► 视频号 / 抖音 ── 竖屏(手动上传,无开放 API)
   └─► X(可选) ── 横/竖均可(API,注意免费层限制)
```

## 0. 一次性准备(只配一次)

### 工具环境
```bash
conda install -y -c conda-forge ffmpeg        # 视频合成
pip install edge-tts numpy scipy              # 配音 + BGM 合成
pip install google-api-python-client google-auth-oauthlib google-auth-httplib2  # YouTube
pip install tweepy                            # X(可选)
# Node 由 skill scaffold 带,后面 npm install 即可
```

### 发布凭证(集中放 `~/.config/video-publish/`,换目录不丢)
- **YouTube**:console.cloud.google.com 建项目 → 启用 *YouTube Data API v3* → 凭据建 *OAuth 客户端 ID(桌面应用)* → 下载 JSON 改名 `client_secret.json` 放进该目录;OAuth 同意屏幕把自己的频道账号加成「测试用户」。
- **X(可选)**:developer.x.com 建 App,User auth 设 *Read and Write*,改权限后**重新生成** Access Token,四个密钥写进 `~/.config/video-publish/x.env`(模板见 `~/articles/.claude/skills/publish-video/assets/x.env.example`)。

> YouTube 首次上传会弹一次浏览器授权,之后 token 缓存、无感。

## 1. 阶段一:文章 → 视频(report-to-video)

### 1.1 建报告目录
```bash
DIR=~/vllm/modeldoctor/.experiments/wechat-reports/<reportId>
mkdir -p $DIR/anim && cp -r ~/articles/.claude/skills/report-to-video/assets/remotion/* $DIR/anim/
cp ~/articles/.claude/skills/report-to-video/assets/scripts/*.py $DIR/
cd $DIR/anim && npm install          # 拉 Remotion + 内置 Chromium(~2min)
```

### 1.2 选主题
`anim/src/theme.ts` 顶部 `MODE = 'dark'`(极客深色)或 `'light'`。

### 1.3 写分镜(每个故事节拍一个场景)
- 改 `anim/src/*.tsx`(worked example 是 7 个场景:Hero/Topology/PrefixTree/RunData/BarRace/NoHotspot/Outro)。
- **每个场景 orientation-aware**:`const land = width > height`,竖屏纵向堆叠、横屏两栏。
- **竖屏必须留视频号/抖音安全区**:内容控制在 `y ∈ [~220, ~1410]`(顶 ~220 避开状态栏+标签页,底 ~510 留给账号行/操作按钮)。
- 数字用 `interpolate` 计数、入场用 `spring`、随机用 `random(seed)`(**不要 Math.random**)。
- 单帧快速验证,先看图再渲染:
```bash
cd $DIR/anim && npx remotion still src/index.ts <Comp> out/chk.png --frame=<n>
```

### 1.4 渲染分镜(竖屏)
```bash
for c in Hero Topology PrefixTree RunData BarRace NoHotspot Outro; do
  lc=$(echo $c|tr A-Z a-z)
  npx remotion render src/index.ts $c out/$lc.mp4
done
```

### 1.5 配音 + BGM + 合成
- 改 `build_video.py` 里的 `SHOTS`:`(场景名, 中文口播)`。**口播按 TTS 改写**——数字读中文(`百分之二十二`)、字母拆开(`K V`、`P 九五`)。
- 生成无版权 BGM:`python make_bgm.py .`
- 合成(默认 **竖屏、无字幕**):
```bash
cd $DIR
python build_video.py .            # → video-portrait.mp4
python build_video.py . --subs     # 要烧字幕(默认不烧,见下方说明)
```

> **默认竖屏、无字幕**是有意为之:视频号/抖音 muted 自动播,画面自带文案;字幕会被平台底部 UI 遮挡。横屏/字幕都需显式开启。

## 2. 阶段二:发布(publish-video)

### 2.1 写元数据 `publish_meta.json`(放视频同目录)
```json
{
  "youtube_title": "标题(横屏正片别带 #Shorts)",
  "youtube_desc": "简介,多行 OK",
  "youtube_tags": ["vLLM", "Higress", "..."],
  "x_text": "X 文案,≤280 字符更稳"
}
```
脚本自动读它;也能用 `--title / --desc-file / --tags / --text` 临时覆盖。

### 2.2 YouTube(发横屏正片)
先把横屏版渲出来:
```bash
cd $DIR/anim
for c in Hero Topology PrefixTree RunData BarRace NoHotspot Outro; do
  lc=$(echo $c|tr A-Z a-z); npx remotion render src/index.ts ${c}L out/${lc}_l.mp4
done
cd $DIR && python build_video.py . --land     # → video-landscape.mp4
```
上传 → 验收 → 公开:
```bash
PV=~/articles/.claude/skills/publish-video/assets/scripts
python $PV/upload_youtube.py video-landscape.mp4 --privacy unlisted   # 先不公开自己看
python $PV/set_privacy.py <videoId> public                            # 确认后公开
# 改回/删除:  set_privacy.py <id> unlisted | private | delete
```

### 2.3 视频号 / 抖音(发竖屏)
`video-portrait.mp4` **手动上传**(这两个平台没有放开的视频发布 API)。简介用 `publish_meta.json` 里的文案。竖屏不要也发到 YouTube(同内容重复)。

### 2.4 X(可选)
```bash
python ~/articles/.claude/skills/publish-video/assets/scripts/upload_x.py video-portrait.mp4
```
⚠️ X 免费层大概率不含视频上传,报 403/453 就得升 Basic($200/月)。

## 3. 常见坑

| 现象 | 原因 / 解法 |
|---|---|
| 竖屏标题/字幕被平台 UI 遮 | 没留安全区;竖屏内容收进 `y∈[220,1410]`,字幕(若开)MarginV~430 |
| YouTube 改公开报 **403 insufficient scopes** | upload token 权限不够;`set_privacy.py` 用更高的 force-ssl 权限,首次再授权一次 |
| 浏览器提示「Google 未验证此应用」 | 自己的测试 app,正常,点 **Continue** |
| 字幕每行行首多个逗号 | `.ass` 的 `Format:` 行漏了 `Name` 字段 |
| 字幕巨大且在画面中间 | libass 用了 288px 参考系;`.ass` 写死 `PlayResX/Y`(build 脚本已处理) |
| 横屏布局错位、内容跑到屏外 | 场景没做 orientation-aware;用 `useVideoConfig()` 分支 |
| Outro 二维码扫不出 | 深色主题下二维码外框要保持白色 `#fff` |
| 配音比动画长、结尾卡住 | Composition 时长设得比口播长;build 脚本用 tpad 兜底 |
