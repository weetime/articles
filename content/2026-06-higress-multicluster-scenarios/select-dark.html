同一套 Higress 能力
按场景落成 4 种方案

决策只有两根轴:主集群要不要自己跑模型(serving) · ai-route 策略放主集群入口还是下沉子集群。两轴一交叉,落成下面四套。
四套方案 能力栈相同,差异只在拓扑与策略位置
单地域起步单集群
一套 Higress 管控 + 算力同体:ai-route 原生选路 → ai-provider → 本地模型。无跨集群、无回环、不破环。
选它当最小 MVP / 内部试点 / 单地域起步,暂不需要跨集群容灾
多集群标准管控算力分离★ 推荐
主集群 纯路由不 serving(主备 HA 永远在线,CR + 共享 Redis 同步);子集群跑模型,跨集群上游池 SE+DR 加权/健康/会话分发。无回环、无需破环。
选它当多集群常态化,要集中管控 + 加权路由 + 自动故障转移
算力复用管控算力一体
主集群既路由又自己跑模型(回环),上游池端点含主集群自身;回环腿需 model 改写破环,治理再过一遍。
选它当资源紧张 / 集群数少,愿复用主集群 GPU、接受破环复杂度
跨地域全量去中心
每集群全量模型 + 全套 Higress,F5 直连就近、无中心路由跳;策略下沉各集群自治,仅本地异常时兜底互转。代价是全量复制。
选它当多地域就近 + 全量容灾,容忍复制的存储与一致性成本
逐维对比 同一能力栈,差异集中在这几维
维度 单集群 管控算力分离 ★ 管控算力一体 去中心
集群数1主 + 子主 + 子多 · 对等
主集群角色管控+算力纯路由路由+算力(回环)无中心
ai-route 策略本集群主集群入口主集群入口下沉子集群
回环 / 破环不涉及无有 · model 破环无
模型分布全在本集群子集群承载主 + 子每集群全量
跨集群上游池不需要SE + DRSE + DR(含回环)可选互转
复杂度 / 适用低 · 起步中 · 标准首选中高 · 算力复用高 · 跨地域全冗余
